theravadan.org

Goal

To create a website with no javascript. This was achieved in a manner of speaking...

Why?

For low maintenance and low overhead on server and browser leading to hopefully quicker access times. Also to allow people to download the entire website and either browse locally or use file search.

Strictly Speaking...

All the pages are served statically, meaning that the webpages, as seen in the browser, are located in individual, self-contained files on a server.

The only exception is the search page which uses anvilworks a regime of creating websites with only python. Strictly speaking, the search page contains mechanisms to interpret the python code into javascript.

Sutta Central

Bhikkhu Sujato has worked hard to bring the pāli texts to a global audience with his website suttacentral. Other pāli scholars and Buddhist monks have since joined him and also placed their translation work on bilara.

This website used those translations to create a static website with keyword search.

It is not meant to replace suttacentral and it cannot, suttacentral is complete. But the words of Lord Buddha are valuable and so are the translations. To place another copy of these words on the internet hopefully will increase the odds of them being discovered.

Technical

The pāli texts and translations take currently around 700,000 lines of text. These are divided into small units called sutta that can be served individually from one html file.

A small python program was used to convert the json formatted files on bilara into a mark-up language called restructuredText for easier editing and formatting.

The restructuredText files are then converted into html files using pelican and uploaded to a static hosting company.

Search

With no javascript and no code running on a server anywhere, creating a search function is problematic. The solution was to use a postgresql database which has state-of-the-art text search capabilities. Fortunately, postgresql search is easy to implement without much investment.

postgrest was then used for interface between the database and web using a REST API.

Lastly, the search page. It is simple enough, only one input box. Use javascript? Well... this website's IT already had an account on anvilworks so that seemed an easier route...

To allow the reader to search, there has to be a page that queries the database and places the research results into a formatted page. Using only python on anvilworks was an easy implementation. Probably in the future, the search page will be programmed with javascript alone.
